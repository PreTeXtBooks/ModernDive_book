    Section 10.5 Theory-based inference for multiple linear regression

    In this section, we introduce the conceptual framework for inference in multiple linear regression. Inference for multiple linear regression is a natural extension of inference for simple linear regression. Recall the linear model for the \(i\)th observation:
    🔗
    \begin{equation*} y_i = \beta_0 + \beta_1 \cdot x_{i1} + \dots + \beta_p x_{ip} + \epsilon_i. \end{equation*}
    We again assume the error term is normally distributed: \(\epsilon_i \sim \text{Normal}(0, \sigma)\text{.}\) It follows that:
    🔗
    \begin{equation*} y_i \sim \text{Normal}(\beta_0 + \beta_1 \cdot x_{i1} + \dots + \beta_p x_{ip},\, \sigma) \end{equation*}
    for \(i=1,\dots,n\text{.}\) The least-squares estimators \(b_0, b_1, \dots, b_p\) are linear combinations of normally distributed random variables, so they are also normally distributed. The standard deviation of each estimator \(b_j\) is a function of \(\sigma\) and the observed data; when we estimate \(\sigma\) with \(s\text{,}\) the resulting estimated standard deviation is called the standard error \(SE(b_j)\text{.}\) The regression table produced by get_regression_table() contains these standard errors:
    🔗
    get_regression_table(mod_mult)

    Subsection 10.5.1 Model dependency of estimators

    The most important difference between multiple and simple linear regression inference is that the least-squares estimators represent partial slopes whose values depend on the other regressors in the model. If we change the set of regressors, the least-squares estimates and their standard errors will likely change as well, leading to different confidence intervals, different test statistics, and potentially different conclusions.
    🔗
    Using the coffee_data example, suppose we consider a model with only three numerical regressors (dropping continent_of_origin):
    🔗
    # Model with three numerical regressors:
mod_mult_1 <- lm(
  total_cup_points ~ aroma + flavor + moisture_percentage,
  data = coffee_data)
coef(mod_mult_1)
sigma(mod_mult_1)
    Now consider a further reduced model without the regressor flavor:
    🔗
    # Model without flavor:
mod_mult_2 <- lm(
  total_cup_points ~ aroma + moisture_percentage, data = coffee_data)
coef(mod_mult_2)
sigma(mod_mult_2)
    The partial slope for aroma in mod_mult_1 differs from the partial slope for aroma in mod_mult_2 because the models are different. Similarly, every other coefficient, the standard deviation estimate, and inferential results can differ. Any results and interpretations are only valid for the model used — never assume a conclusion from one model translates to a different model.
    🔗
    🔗

    Subsection 10.5.2 Confidence intervals

    A 95% confidence interval for any coefficient in multiple linear regression is constructed in exactly the same way as for simple linear regression: \(b_j \pm q \cdot SE(b_j)\text{,}\) where the critical value \(q\) is determined by the confidence level and degrees of freedom \(n - p\text{.}\)
    🔗
    For the coffee example with model mod_mult: three numerical regressors, one four-level factor (three dummy variables), and an intercept give \(p = 7\) coefficients. The 95% CI for the partial slope of aroma (\(\beta_1\)) is:
    🔗
    \begin{equation*} b_1 \pm q \cdot SE(b_1). \end{equation*}
    We can compute 98% confidence intervals for all coefficients by passing conf.level = 0.98 to get_regression_table():
    🔗
    get_regression_table(mod_mult, conf.level = 0.98)
    The interpretation for the flavor coefficient, for example, is: “We are 98% confident that the value of \(\beta_2\) (the population partial slope for flavor) is between the reported lower and upper bounds.”
    🔗
    🔗

    Subsection 10.5.3 Hypothesis test for a single coefficient

    The hypothesis test for one coefficient, say \(\beta_j\text{,}\) is similar to that for simple linear regression. The general two-sided test is:
    🔗
    \begin{align*} H_0&: \beta_j = 0 \quad \text{with all other parameters given and arbitrary}\\ H_A&: \beta_j \ne 0 \quad \text{with all other parameters given and arbitrary.} \end{align*}
    In simple linear regression, testing \(\beta_1 = 0\) was testing whether there was a linear relationship between the response and the only regressor. Now, testing \(\beta_j = 0\) is testing whether the corresponding regressor should be part of a linear model that already contains all the other regressors.
    🔗
    For example, using model mod_mult_1 and testing \(\beta_2 = 0\) (the partial slope for flavor):
    🔗
    get_regression_table(mod_mult_1)
    The \(t\)-test statistic is \(t = b_2 / SE(b_2)\text{;}\) the associated \(p\)-value is near zero, so we reject \(H_0\) and conclude that flavor is statistically significant when added to a model that already contains aroma and moisture_percentage. By contrast, moisture_percentage is not statistically significant in that model, suggesting we can remove it.
    🔗
    🔗

    Subsection 10.5.4 Hypothesis test for model comparison

    There is another hypothesis test for multiple linear regression: comparing a full model with a given set of regressors to a reduced model that uses only a subset of those regressors. This test is called an ANOVA test or \(F\)-test, because the test statistic follows an \(F\) distribution.
    🔗
    Using coffee_data, compare the full model mod_mult_1 (aroma + flavor + moisture_percentage) to the reduced model mod_mult_2 (aroma + moisture_percentage):
    🔗
    \begin{align*} H_0&: \text{the reduced model is adequate}\\ H_A&: \text{the full model is needed.} \end{align*} 
    anova(mod_mult_2, mod_mult_1)
    The small \(p\)-value leads us to reject the null hypothesis and conclude that the full model (including flavor) was needed.
    🔗
    The ANOVA test can test more than one regressor at a time, which is especially useful for factors. To test whether continent_of_origin should be included, compare mod_mult_1 (without the factor) to mod_mult (with the factor):
    🔗
    anova(mod_mult_1, mod_mult)
    We reject the null hypothesis and conclude that it is appropriate to add continent_of_origin to a model that already has aroma, flavor, and moisture_percentage. When testing factors in ANOVA, all factor levels must always be tested simultaneously.
    🔗
    🔗

    Subsection 10.5.5 Model fit and diagnostics

    As for simple linear regression, we use residuals to determine the fit of the model and whether assumptions are met. In particular, the plot of residuals against the fitted values should look like a null plot. In multiple regression, when the residual-vs-fitted plot shows a pattern, we know that at least one assumption is violated, though it is no longer immediately clear which one.
    🔗
    Using the coffee example, we have shown that aroma, flavor, and continent_of_origin were statistically significant but moisture_percentage was not. We create a final model mod_mult_final with only the relevant regressors and examine residuals:
    🔗
    # Fit final model:
mod_mult_final <- lm(total_cup_points ~ aroma + flavor + continent_of_origin,
                     coffee_data)
# Get fitted values and residuals:
fit_and_res_mult <- get_regression_points(mod_mult_final)

    g1 <- fit_and_res_mult |>
  ggplot(aes(x = total_cup_points_hat, y = residual)) +
  geom_point() +
  labs(x = "fitted values (total cup points)", y = "residual") +
  geom_hline(yintercept = 0, col = "blue")
g2 <- ggplot(fit_and_res_mult, aes(sample = residual)) +
  geom_qq() +
  geom_qq_line(col = "blue", linewidth = 0.5)
grid.arrange(g1, g2, ncol = 2)
    The plot of residuals against fitted values (left) appears to be close to a null plot, supporting the Linearity condition. The Equal or constant variance also holds as the vertical dispersion is fairly uniform. Since we assumed the data were collected randomly with no time sequence, the assumption of Independence is acceptable. The QQ-plot suggests (with the exception of one or two observations) that the residuals follow approximately the normal distribution. We can conclude that this model holds the assumptions well.
    🔗

    Exercises Exercises

    1.
    Why is it essential to know that the estimators \(b_0, b_1, \dots, b_p\) in multiple linear regression are unbiased?
    🔗
    1. It ensures that the variance of the estimators is always zero.
      🔗
      🔗
    2. It means that, on average, the estimators will equal the true population parameters they estimate.
      🔗
      🔗
    3. It implies that the estimators have a standard error of zero.
      🔗
      🔗
    4. It suggests that the regression model will always have a perfect fit.
      🔗
      🔗
    Answer.
    B. Unbiasedness means that over many random samples, the average value of each estimator equals the true population parameter.
    🔗
    🔗
    🔗
    2.
    Why do the least-squares estimates of coefficients change when different sets of regressors are used in multiple linear regression?
    🔗
    1. Because the coefficients are recalculated each time, irrespective of the regressors.
      🔗
      🔗
    2. Because the residuals are always zero when regressors are changed.
      🔗
      🔗
    3. Because the value of each coefficient depends on the specific combination of regressors included in the model.
      🔗
      🔗
    4. Because all models with different regressors produce identical estimates.
      🔗
      🔗
    Answer.
    C. Each partial slope coefficient represents the contribution of its regressor after accounting for all others; changing which others are included changes that contribution.
    🔗
    🔗
    🔗
    3.
    How is a 95% confidence interval for a coefficient in multiple linear regression constructed?
    🔗
    1. By using the point estimate, the critical value from the \(t\)-distribution, and the standard error of the coefficient.
      🔗
      🔗
    2. By taking the standard deviation of the coefficients only.
      🔗
      🔗
    3. By resampling the data without replacement.
      🔗
      🔗
    4. By calculating the mean of all the coefficients.
      🔗
      🔗
    Answer.
    A. A 95% CI is \(b_j \pm q \cdot SE(b_j)\text{,}\) where \(q\) is the critical value from the \(t\)-distribution with \(n - p\) degrees of freedom.
    🔗
    🔗
    🔗
    4.
    What does the ANOVA \(F\)-test for comparing two models in multiple linear regression evaluate?
    🔗
    1. Whether all regressors in both models have the same coefficients.
      🔗
      🔗
    2. Whether the reduced model is adequate or if the full model is needed.
      🔗
      🔗
    3. Whether the residuals of the two models follow a normal distribution.
      🔗
      🔗
    4. Whether the regression coefficients of one model are unbiased estimators.
      🔗
      🔗
    Answer.
    B. The ANOVA \(F\)-test compares the sum of squared residuals of the full and reduced models to determine whether the additional regressors in the full model provide a significantly better fit.
