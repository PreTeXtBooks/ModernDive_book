    Section 7.1 First activity: red balls

    Subsection 7.1.1 The proportion of red balls in the bowl

    Imagine you have a large bowl filled with red and white balls, as shown in Figure 7.1.1. The bowl contains a mix of colors and you want to know what proportion of the balls are red.
    🔗
    The bowl is represented by the bowl data frame in the moderndive package. Let’s explore it:
    🔗
    bowl
    To compute the proportion of red balls, we can first create an indicator variable that is TRUE if a ball is red and FALSE otherwise:
    🔗
    bowl |>
  mutate(is_red = (color == "red"))
    Next, we can count the number of red balls by summing the indicator variable (since TRUE counts as 1 and FALSE as 0):
    🔗
    bowl |>
  mutate(is_red = (color == "red")) |>
  summarize(num_red = sum(is_red))
    The sum() function effectively counts red balls. There are 375 red balls in the bowl. We can compute the proportion by using mean() instead of sum():
    🔗
    bowl |>
  mutate(is_red = (color == "red")) |>
  summarize(prop_red = mean(is_red))
    Using mean() on a TRUE/FALSE variable gives the proportion directly because it computes the fraction of TRUE values. We can write this more concisely as:
    🔗
    bowl |>
  summarize(prop_red = mean(color == "red"))
    The proportion of red balls is 375/2400 = 0.375, or approximately 37.5% of all balls in the bowl.
    🔗
    🔗

    Subsection 7.1.2 Manual sampling

    In practice, it is often impractical or impossible to count every element in a large population. For example, if the bowl contained millions of balls, exhaustively counting all of them would be prohibitive. Instead, we use a physical sampling approach: we insert a shovel into the bowl, draw out a sample, and use it to estimate the population proportion.
    🔗

    Subsubsection 7.1.2.1 One sample

    We insert a shovel into the bowl to collect a sample of balls, as shown in Figure 7.1.2 and Figure 7.1.3.
    🔗
    The shovel has a 5-row by 10-column grid, so it scoops exactly \(5 \times 10 = 50\) balls at a time. In one particular sample, 17 of the 50 balls were red, giving a sample proportion of \(17/50 = 0.34\text{.}\) This is somewhat close to the true population proportion of 0.375, but not exactly equal. The difference arises from sampling variation.
    🔗
    🔗

    Subsubsection 7.1.2.2 Thirty-three samples

    We can repeat this process: return the balls to the bowl, mix them up, and draw another sample. By repeating this many times we can study how the sample proportion varies from sample to sample. Thirty-three students performed this activity, each taking one sample of 50 balls.
    🔗
    The results of all 33 students’ samples are stored in the tactile_prop_red data frame in the moderndive package:
    🔗
    tactile_prop_red
    Let’s visualize the distribution of sample proportions with a histogram:
    🔗
    ggplot(tactile_prop_red, aes(x = prop_red)) +
  geom_histogram(binwidth = 0.05, boundary = 0.4, color = "white") +
  labs(x = "Proportion of red balls in each sample",
       title = "Histogram of 33 proportions")
    Notice that no two students got exactly the same proportion. This is sampling variation: different random samples from the same population produce different sample statistics. The distribution of these sample proportions is called the sampling distribution of the sample proportion.
    🔗
    Let’s introduce some key vocabulary:
    • Sampling: The act of drawing a subset (sample) from a larger group (population).
      🔗
      🔗
    • Sampling with replacement: Each selected item is returned to the population before the next draw, so the same item could theoretically be selected more than once.
      🔗
      🔗
    • Random samples: Samples drawn such that every member of the population has an equal chance of being selected.
      🔗
      🔗
    • Sampling variation: The natural variability in a sample statistic from sample to sample.
      🔗
      🔗
    🔗
    Exercises Exercises
    1.
    Why is it important to mix the balls in the bowl before we take a new sample?
    🔗
    Answer.
    To randomize selection and reduce dependence or bias between samples.
    🔗
    🔗
    🔗
    2.
    Why is it that students did not all have the same sample proportion of red balls?
    🔗
    Answer.
    Sampling variation. Different random samples from the same population yield different statistics.
    🔗
    🔗
    🔗
    🔗
    🔗
    🔗

    Subsection 7.1.3 Virtual sampling

    Instead of physically taking samples from a bowl, we can use a computer to simulate the process. This extends the tactile sampling activity to simulations that are faster, scalable, and reproducible. We use the rep_slice_sample() function from the moderndive package.
    🔗

    Subsubsection 7.1.3.1 One virtual sample

    Let’s take one virtual sample of 50 balls from the bowl:
    🔗
    virtual_shovel <- bowl |>
  rep_slice_sample(n = 50)
virtual_shovel
    Now compute the proportion of red balls in this virtual sample:
    🔗
    virtual_shovel |>
  summarize(prop_red = mean(color == "red"))
    🔗

    Subsubsection 7.1.3.2 Thirty-three virtual samples

    We can simulate 33 samples at once by specifying reps = 33:
    🔗
    virtual_samples <- bowl |>
  rep_slice_sample(n = 50, reps = 33)
virtual_samples
    Now we compute the sample proportion for each of the 33 replications using group_by() and summarize():
    🔗
    virtual_prop_red <- virtual_samples |>
  group_by(replicate) |>
  summarize(prop_red = mean(color == "red"))
virtual_prop_red
    We can also do this in one pipeline:
    🔗
    virtual_prop_red <- bowl |>
  rep_slice_sample(n = 50, reps = 33) |>
  summarize(prop_red = mean(color == "red"))
virtual_prop_red
    Visualizing the distribution of the 33 virtual sample proportions:
    🔗
    ggplot(virtual_prop_red, aes(x = prop_red)) +
  geom_histogram(binwidth = 0.05, boundary = 0.4, color = "white") +
  labs(x = "Sample proportion",
       title = "Histogram of 33 sample proportions")
    Exercises Exercises
    1.
    Why couldn’t we study the effects of sampling variation when we used the virtual shovel only once?
    🔗
    Answer.
    One sample shows only one outcome. Many samples are needed to observe variability and form a sampling distribution.
    🔗
    🔗
    🔗
    🔗
    🔗

    Subsubsection 7.1.3.3 One thousand virtual samples

    To better understand the shape of the sampling distribution, let’s take 1000 virtual samples instead of just 33:
    🔗
    virtual_prop_red <- bowl |>
  rep_slice_sample(n = 50, reps = 1000) |>
  summarize(prop_red = mean(color == "red"))
virtual_prop_red

    ggplot(virtual_prop_red, aes(x = prop_red)) +
  geom_histogram(binwidth = 0.04, boundary = 0.4, color = "white") +
  labs(x = "Sample proportion", title = "Histogram of 1000 sample proportions")
    With 1000 samples, the distribution is clearly symmetric and bell-shaped, closely approximated by a normal distribution. This is the sampling distribution of the sample proportion for \(n = 50\text{.}\)
    🔗
    Exercises Exercises
    1.
    Why did we not take 1000 samples of 50 balls by hand?
    🔗
    Answer.
    It is impractical and time consuming. Simulation is fast, scalable, and reproducible.
    🔗
    🔗
    🔗
    2.
    Looking at the histogram, would you say that sampling 50 balls where 30% of them were red is likely or not? What about 10%?
    🔗
    Answer.
    30% is plausible though less common. 10% is very unlikely.
    🔗
    🔗
    🔗
    🔗
    🔗

    Subsubsection 7.1.3.4 Different sample sizes

    What happens to the sampling distribution when we change the sample size? Instead of always using a shovel that scoops 50 balls, consider using shovels of three different sizes, as shown in Figure 7.1.11.
    🔗
    The following code computes 1000 sample proportions for each of three sample sizes (25, 50, and 100) and displays the resulting histograms:
    🔗
    # Segment 1: sample size = 25 ------------------------------
# 1.a) Compute sample proportions for 1000 samples, each sample of size 25
virtual_prop_red_25 <- bowl |>
  rep_slice_sample(n = 25, reps = 1000) |>
  summarize(prop_red = mean(color == "red"))

# 1.b) Plot a histogram to represent the distribution of the sample proportions
ggplot(virtual_prop_red_25, aes(x = prop_red)) +
  geom_histogram(binwidth = 0.05, boundary = 0.4, color = "white") +
  labs(x = "Proportion of 25 balls that were red", title = "25")


# Segment 2: sample size = 50 ------------------------------
# 2.a) Compute sample proportions for 1000 samples, each sample of size 50
virtual_prop_red_50 <- bowl |>
  rep_slice_sample(n = 50, reps = 1000) |>
  summarize(prop_red = mean(color == "red"))

# 2.b) Plot a histogram to represent the distribution of the sample proportions
ggplot(virtual_prop_red_50, aes(x = prop_red)) +
  geom_histogram(binwidth = 0.05, boundary = 0.4, color = "white") +
  labs(x = "Proportion of 50 balls that were red", title = "50")


# Segment 3: sample size = 100 ------------------------------
# 3.a) Compute sample proportions for 1000 samples, each sample of size 100
virtual_prop_red_100 <- bowl |>
  rep_slice_sample(n = 100, reps = 1000) |>
  summarize(prop_red = mean(color == "red"))

# 3.b) Plot a histogram to represent the distribution of the sample proportions
ggplot(virtual_prop_red_100, aes(x = prop_red)) +
  geom_histogram(binwidth = 0.05, boundary = 0.4, color = "white") +
  labs(x = "Proportion of 100 balls that were red", title = "100")
    Comparing the three histograms in Figure 7.1.12, we observe three important features:
    1. All three histograms are centered around the same value, slightly below 0.4, close to the true population proportion of 0.375.
      🔗
      🔗
    2. All three histograms are somewhat bell-shaped.
      🔗
      🔗
    3. As the sample size increases from 25 to 50 to 100, the sampling variation decreases: the histograms become narrower. Larger samples produce sample proportions that are more consistently close to the true population proportion.
      🔗
      🔗
    🔗
